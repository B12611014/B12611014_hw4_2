🏺 《失落文明：語言學家》

—— LLM 多任務互動式文本冒險遊戲

本專案是一款以「古文明」為主題的互動式文字冒險遊戲，結合大語言模型（LLM）進行摘要、關鍵句擷取、事件抽取、推理、角色對話 等任務。玩家在闖關過程中逐步揭開古文明遺跡背後的秘密。

本遊戲共五章，每一章都透過不同的 AI 任務引導玩家思考、探索，並輸出分析結果到 /output/ 目錄。

⸻

📖 1. 遊戲描述（Game Description）

你扮演一名進入古代遺跡探險的冒險者，手中只有破碎的筆記與遺留下的文本線索。
每一章節包含不同的考驗，你需要依靠 AI 技術處理文本、分析事件、推理隱藏訊息，再與古文明的「智慧意識」對話，獲得密碼提示以解鎖遺跡的下一道封印。

🎮 遊戲章節概覽

章節	主題	        主要任務
第1章	失落遺跡入口     整理解讀古文卷內容（摘要）
第2章	地底洞窟	    擷取重要線索（關鍵句抽取）
第3章	神秘廳堂	    找出事件與時間排序
第4章	寶庫外層封印	 推理寶庫秘密 + 古文明智慧對話
第5章	最終寶庫	    推理文明核心價值 + 終局密碼

玩家需輸入密碼才能進入下一章，並在過程中學習到不同文本分析技巧。

⸻

🧠 2. 使用到的 LLM 任務（LLM Tasks Applied）

本專案示範了多種對文本非常常用的 LLM 任務：

① 摘要（Summarization）

將大量而雜亂的古文明筆記整理成易懂的核心內容。

② 關鍵句子抽取（Key Sentence Extraction）

找出文本中最有價值、最具意義的句子，協助玩家理解線索。

③ 事件與時間抽取（Event Extraction）

從文本中抓出：
	•	事件（event）
	•	時間（time）
	•	時序 (sequence)

並輸出成列表，讓玩家能追蹤古文明歷史。

④ 因果推理（Causal Reasoning）

從事件推敲：
	•	為何文明會滅亡？
	•	為何寶庫被封印？
	•	遺跡留下的核心訊息是什麼？

這部分讓遊戲具有「解謎」感。

⑤ 對話（Dialogue Generation）

包含兩種角色對話：
	•	第 4 章：與寶庫守護者交談，獲得提示
	•	第 5 章：與古文明智慧核心對話，理解他們的文化與價值

⑥ 玩家互動輸入

玩家會輸入密碼：
	•	第4章：智慧（wisdom）
	•	第5章：傳承、珍惜、延續

🔍 3. 我的發現（My Discovery）

在這次作業的遊戲設計與測試過程中，我對大語言模型（LLM）在遊戲化文本處理、互動式故事設計以及 API 操作上有了更深入的理解。

首先，我發現 LLM 非常擅長處理破碎、不連續的文本。即使資料像古文明筆記那樣結構鬆散、語句片段化，模型仍能透過摘要、關鍵句抽取、事件分析與推理等任務，重組並萃取出有意義的資訊。這讓我能將大量原始文本交給模型處理，而玩家只需關注劇情本身，使遊戲流程更流暢。

其次，多種 LLM 任務的結合 使整體遊戲更有深度。每一章使用不同的功能——像關鍵句擷取、事件排序、原因推理、角色對話、隱性提示生成等等。這些任務讓遊戲不再只是純聊天，而是透過 AI 的文字挖掘能力，引導玩家逐步拼湊古文明的故事脈絡。這樣的組合讓我體會到 LLM 不只是用來回答問題，而是可以成為「敘事引擎」。

在遊戲化設計上，我也發現將 AI 任務包裝成關卡機制能創造新的體驗。原本艱澀的 NLP 任務，如事件抽取或推理，只要轉換成「闖關」的形式，玩家的接受度就明顯提升。這讓我理解 AI 與遊戲設計的結合具備很大的創意空間，可以讓學習與互動變得自然又有趣。

此外，在幫 AI 安排角色時——例如第 4、5 章的文明守護者、古代意識、智慧庫管理者——我發現 LLM 其實可以成為遊戲世界觀的一部分，而不只是工具。它可以「扮演」古文明的智慧，回應風格也可以透過 prompt 明顯調整，這是我過去只是用 ChatGPT 聊天時沒有特別體會到的。

最後，在這次專案中使用 OpenAI API 也帶給我新的感受。相比於直接使用 ChatGPT 介面，透過 API：
	•	我可以完整控制 prompt、模型設定、溝通流程
	•	可以讓 AI 參與不同章節的劇情運算
	•	可以把 LLM 當作遊戲系統的一環，而非單一對話視窗
	•	可以將輸出寫入檔案、做後處理、與其他程式模組整合

這讓我第一次感受到 LLM 不只是聊天工具，而是一個可嵌入遊戲或應用的 AI 引擎。

這個專案讓我更理解 LLM 在文本推理、互動敘事設計、任務組合、系統整合上的潛力，也讓我體會到使用 OpenAI API 開發與使用 ChatGPT 的差別。對我來說，ChatGPT 是「人與 AI 對話」，而 API 是「AI 成為程式的一部分」。

很開心有這個機會藉由課程作業接觸到Openai API，讓我獲益良多！

▶️ 5. 遊戲啟動方式

Step 1. 安裝套件

pip install openai

Step 2. 將你的 OpenAI API Key 放入 config.json

{
    "api_key": "YOUR_KEY_HERE",
    "model": "gpt-3.5-turbo"
}

Step 3. 執行遊戲

python game_main.py
